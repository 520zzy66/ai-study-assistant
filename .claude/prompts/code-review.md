# 代码审查与修复

对当前代码（或本次变更）进行全面审查，发现并修复问题。

---

## 审查方法论

在逐维度检查前，先掌握以下四种核心方法：

### 1. 数据链路追踪
追踪数据从写入到读取的完整链路，不要只看单个方法。
- [ ] 写入时存的格式/路径，读取时是否一致？
- [ ] 跨方法调用时，参数是否正确传递？
- 案例：upload 存相对路径，delete 也用相对路径 → 永远删不到文件

### 2. 配置消费链验证
配置项绑定了 Bean ≠ 真正生效，必须验证是否被实际消费。
- [ ] `@ConfigurationProperties` 字段是否在业务代码中被读取？
- [ ] `application.yml` 配置项是否传递给了底层客户端？
- 案例：`ai.chat.timeout` 绑定了 AiChatProperties，但 ChatClient 构建时从未使用

### 3. 安全攻击视角
不只看"能不能跑"，想"攻击者怎么绕过"。
- [ ] 防护机制是否有时间窗口漏洞？
- [ ] 限流/锁定策略，慢速攻击能否绕过？
- 案例：每次失败都重置 TTL → 慢速爆破永不触发锁定

### 4. 交叉验证
定义了常量/工具类，要验证是否被所有使用处引用。
- [ ] Constants 中的常量是否在所有使用处被引用？
- [ ] 提取的公共方法，旧代码是否遗漏引用？
- 案例：Constants.LOGIN_MAX_ATTEMPTS=5，但 LoginAttemptServiceImpl 硬编码 MAX_ATTEMPTS=5

---

## 审查维度

每个维度至少找出一条改进点（无问题则标记 ✅）。

### 1. 安全性（Security）
- [ ] 认证/授权是否有绕过风险？
- [ ] 敏感端点是否被误暴露？（如 `/actuator/env`）
- [ ] HTTP 状态码是否与语义一致？
- [ ] 用户输入是否校验和过滤？
- [ ] SQL 注入、XSS、路径遍历是否防护？
- [ ] 数据库保留字是否正确转义？（如 `user`、`order`）
- [ ] 防暴力破解是否有时间窗口漏洞？

### 2. 并发安全（Concurrency）
- [ ] 是否存在 TOCTOU（先查后改）竞态？
- [ ] 是否依赖数据库唯一索引兜底？失败是否有恰当处理？
- [ ] 共享状态是否有适当同步？
- [ ] Redis 操作是否原子？

### 3. 资源管理（Resources）
- [ ] 内存是否有无界增长风险？
- [ ] 流/连接/文件是否正确关闭？
- [ ] 批量操作是否使用批量 API？

### 4. 异常处理（Error Handling）
- [ ] 是否遗漏常见异常类型？
- [ ] 异常信息是否友好且不泄露内部细节？
- [ ] 降级/兜底逻辑是否合理？
- [ ] finally 是否正确清理资源？

### 5. 架构与规范（Architecture）
- [ ] 是否符合三层架构？
- [ ] Controller 是否包含业务逻辑？
- [ ] DTO/VO/Entity 是否分离？
- [ ] 是否存在重复代码？

### 6. 代码质量（Quality）
- [ ] 配置项是否真正被消费？（排查 dead config）
- [ ] 常量/工具类是否被正确引用？（交叉验证）
- [ ] 空指针、越界等边界条件是否处理？
- [ ] 日志级别是否恰当？

---

## 输出格式

```
### 【问题 N】问题简述
**级别**：🔴 严重 / 🟡 建议 / 🔵 可选
**维度**：安全性 / 并发 / 资源 / 异常 / 架构 / 质量
**发现方法**：数据链路追踪 / 配置消费链验证 / 安全攻击视角 / 交叉验证
**影响**：实际影响描述

**建议修改方案**：
具体的修改建议或代码
```

汇总表：
```
| # | 级别 | 维度 | 问题 | 发现方法 | 涉及文件 |
|---|------|------|------|---------|---------|
```

---

## 修复规则

1. 优先修复 🔴 严重问题
2. 分析 → 定位 → 最小修改 → 验证
3. 修复后再次扫描确认无新问题
4. 不进行无关重构
5. 尽量保持现有代码结构不变；
6. 保证修改后项目仍可正常编译运行。

---

## 验证规则

- [ ] 修复后逻辑正确
- [ ] 无新编译/运行时错误
- [ ] 与 `docs/spec.md` 一致

---

## 威胁模型速查

| 自问 | 关注点 |
|------|--------|
| "怎么绕过这个逻辑？" | 认证绕过、越权、注入 |
| "高并发下会怎样？" | 竞态、死锁、数据不一致 |
| "3 个月后集合多大？" | 内存泄漏、无界增长 |
| "客户端能处理这个错误吗？" | HTTP 状态码、消息格式 |
| "写入和读取路径一致吗？" | 数据链路不匹配 |
| "这个配置生效了吗？" | Dead config |
| "慢速攻击能绕过吗？" | TTL/限流漏洞 |
| "常量定义了但引用了吗？" | 交叉验证 |
