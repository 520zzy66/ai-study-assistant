> 适用作品：AI Study Assistant - 基于大模型的个性化资源生成与学习多智能体系统
> 目标赛道：第十五届中国软件杯 A3「基于大模型的个性化资源生成与学习多智能体系统开发」
> 文档用途：指导其他 AI 或设计人员快速生成一份符合初赛提交标准、可用于答辩展示与 7 分钟演示视频脚本联动的高质量 PPT。

## 0. 当前项目状态校准（2026-07-18 执行版）

本规格书应以当前仓库实现为准，PPT 文案必须做到“突出亮点，但不夸大未落地能力”。

### 0.1 已落地、可作为答辩主卖点的能力

- 前端已覆盖首页、资料管理、AI 问答、AI 总结、资源工坊、思维导图、题库、错题本、学习计划、历史记录、用户中心等主要页面。
- 首页已优化为学习闭环入口，卡片可以承接资料、问答、总结、计划、错题、资源工坊等真实功能。
- 资源工坊已接入 `ResourceWorkshop.vue`、`AgentTraceTimeline.vue`、`ResourceAssetPanel.vue`、`PodcastAssetCard.vue`、`ImageAssetGallery.vue` 等组件，适合作为 A3 核心展示入口。
- 后端已存在 `ResourceOrchestratorAgent`、`MultimodalAgent`、`SafetyAgent` 及 `AgentTrace`，资源包生成链路具备多 Agent 编排表达基础。
- 多模态资产已通过 `ResourceAsset`、`ResourceAssetController`、`ResourceAssetService` 形成元数据、存储、下载、重试的闭环。
- 科大讯飞 TTS 与文生图 Provider 已进入后端 Provider 层，配置项通过 `XFYUN_TTS_*`、`XFYUN_IMAGE_*`、`XFYUN_APP_ID` 等环境变量注入，前端不接触密钥。
- Docker / prod 默认大模型链路已切到讯飞星火 OpenAI 兼容接口：`AI_BASE_URL=https://spark-api-open.xf-yun.com/v1`、`AI_CHAT_MODEL=lite`。
- Embedding 已预留/接入 DashScope 方案，旧本地 Ollama 方案不作为比赛 Demo 的默认路径。

### 0.2 可以讲，但必须作为“数据基础/后续扩展”的能力

- `EvaluationAgent`、`RecommendationAgent`、教育大模型深度迁移、智能 PPT/虚拟人讲解等能力，可以放在未来规划页，不要写成当前已完整落地。
- AI 问答主链路仍是 RAG 问答能力，不要描述为已被 TutorAgent 或全新智能体完全替代。
- 防幻觉能力应表述为“RAG 约束 + SafetyAgent 校验 + 不知道时说明”，不要使用“完全无幻觉”。

### 0.3 本轮 PPT 制作策略

- 在没有真实运行截图素材时，PPT 先输出“可替换截图占位版”：所有截图位必须标清 S 编号、目标页面和建议截取内容。
- 架构、流程、评分对应、Demo 路线使用可编辑 PowerPoint 图形表达，避免用不可修改的长图。
- 可直接答辩的页内文案要短，详细说明放到备注讲稿区。
- 最终 PPT 建议输出到 `outputs/ai-study-assistant-a3-competition.pptx`。

## 1. PPT 总体定位

### 1.1 一句话定位

本作品不是普通的 AI 学习问答工具，而是面向高校课程学习场景的「个性化资源生成 + 多 Agent 协同编排 + 多模态学习资产生产」系统。

### 1.2 核心表达

整份 PPT 必须围绕以下主线展开：

```text
学生学习痛点
  -> 个性化学习画像
  -> 多 Agent 资源工坊
  -> 文档、题库、路径、思维导图、多模态音频/图片资源生成
  -> 首页资源推送与学习闭环
  -> 防幻觉、安全、可部署、可持续扩展
```

### 1.3 评委要被记住的三个关键词

1. 多 Agent 协同：ProfileAgent、SummaryAgent、QuizAgent、PlanAgent、MultimodalAgent、SafetyAgent 形成流水线。
2. 个性化资源工坊：基于学生画像、课程资料、学习目标，一键生成资源包。
3. 科大讯飞多模态增强：TTS 知识点播客化、文生图知识配图，为比赛命题企业技术生态做适配。

### 1.4 建议页数

建议制作 20 页，控制在 7 分钟以内讲完。

节奏建议：

| 模块 | 页数 | 时间 |
|---|---:|---:|
| 开场与痛点 | 3 页 | 50 秒 |
| 方案与架构 | 5 页 | 110 秒 |
| 核心功能展示 | 7 页 | 180 秒 |
| 技术创新与安全质量 | 3 页 | 90 秒 |
| 总结与展望 | 2 页 | 50 秒 |

## 2. 视觉风格要求

### 2.1 总体风格

风格关键词：

- 高校教育科技
- 干净、专业、可信
- 具有 AI 产品感，但不要做成纯炫技科技风
- 图表清晰，截图真实，避免堆字

### 2.2 色彩建议

主色：

- 深青绿：`#0F766E`
- 科技蓝：`#2563EB`
- 墨黑：`#111827`

辅助色：

- 浅青绿：`#CCFBF1`
- 浅蓝：`#DBEAFE`
- 中性灰：`#F3F4F6`
- 警示橙：`#F97316`

避免：

- 大面积紫色渐变
- 纯黑赛博风
- 过多花哨装饰
- 与项目现有 UI 风格不一致的卡通风

### 2.3 字体建议

- 标题：思源黑体 / 阿里巴巴普惠体 / Microsoft YaHei Bold
- 正文：Microsoft YaHei / 思源黑体 Regular
- 代码或技术标签：JetBrains Mono / Consolas

字号建议：

- 封面主标题：50-56
- 一级页标题：35-40
- 正文：16-20
- 图注：11-13
- 技术标签：12-14

> 注意：如果使用自动化工具生成 PPT，标题字号不得过小；宁可减少文字，也不要牺牲可读性。

### 2.4 版式规则

- 每页最多 1 个核心观点。
- 每页文字不超过 90 字，技术架构页可适当增加。
- 所有截图必须配 1 句价值说明。
- 架构图优先用横向流程或分层结构。
- 不要使用密集表格堆满页面。

## 第 7 页：多 Agent 架构设计

标题：

```text
从单体调用升级为多 Agent 流水线
```

必须展示的 Agent：

```text
ResourceOrchestratorAgent
  -> ProfileAgent
  -> SummaryAgent
  -> QuizAgent
  -> PlanAgent
  -> MultimodalAgent
  -> SafetyAgent
```

每个 Agent 的一句话职责：

| Agent | 职责 |
|---|---|
| ProfileAgent | 解析学生目标、基础、偏好和学习上下文 |
| SummaryAgent | 生成课程讲解文档和知识组织 |
| QuizAgent | 生成不同类型练习题与解析 |
| PlanAgent | 规划阶段化学习路径 |
| MultimodalAgent | 生成多模态脚本、音频和图片资产 |
| SafetyAgent | 做结构校验、内容安全和质量兜底 |

架构表达：

```text
Controller
  -> ResourcePackageService 门面
  -> ResourceOrchestratorAgent
  -> Specialist Agents
  -> Tools
  -> Existing Services / Xfyun Providers
```

设计亮点：

- 不推倒重来，复用现有稳定 Service。
- Agent 负责决策和分工，Tool 负责执行。
- Trace 可视化展示每一步耗时和状态。

- 使用 S8 Agent Trace 时间线截图。
